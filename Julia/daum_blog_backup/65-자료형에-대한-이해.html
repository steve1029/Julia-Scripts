unsigned 의 뜻은 부호가 없다는 뜻이다. 가령, 정수인 int 의 경우 -4 라는 수를 생각해보자. int 는 4 byte 이므로,

4 byte = 32 bit, 즉 32개의 0,1 로 표현이 된다. 실제로 컴퓨터 상에서 -4 = 0000 0000 0000 0010 로 인식이된다. 

그런데 이는 숫자 4와 같다! 그럼 컴퓨터는 어떻게 4와 -4 를 구분하는 걸까? 

맨첫자리 숫자가 부호를 뜻한다. 0 은 음수, 1은 양수이다. 

즉 4 = 1000 0000 0000 0010 이다. 이렇게 기본적인 자료형들은 맨 첫자리 비트가 부호를 알려주는 의미로 쓰인다.

그래서, signed int 는 값의 표현 범위가 7자리 수로만 표현될 수 있고, 그 범위는 -2^7 ~ 2^7-1, 즉 -128 ~ 127 이 된다.

unsigned 라는 키워드가 없으면 자동으로 signed 로 인식된다.


그럼 unsigned 는 왜 쓰는 걸까? 음수가 나와서는 안되는 자료형에 쓴다. 

가령 키가 160 cm 라면, 키는 음수가 되면 안되므로


unsigned height = 160;


이라고 선언해주는 것이 오류를 줄이는 길이 될 것이다.
