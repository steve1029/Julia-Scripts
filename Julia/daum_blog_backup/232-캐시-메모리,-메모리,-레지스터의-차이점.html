캐시 메모리, 메모리, 레지스터의 차이점

컴퓨터 기본 및 상용 프로그램

2018-01-30 14:48:23

캐시 메모리도 램캐시, 디시크 캐시 등 여러가지가 있지만 아마도 CPU 내부나 메인보드에 설치되는 램캐시를 이야기하시는 것이겠죠? 따라서 본 답변에서 캐시라 함은 램캐시를 지칭하는 것으로 하겠습니다. 또한 캐시와 레지스터는 데이터와 명렬어를 모두 다루지만 답변을 쉽게 하기 위해서 데이터에 대해서만 설명을 하겠습니다.

캐시 란 ?
먼저 캐시란 CPU 에 비해서 상대적으로 속도가 느린 메인메모리로부터 데이터를 불러 들이는 시간을 단축하기 위해서 한번 활용한 데이터의 내용이나 램에서의 위치를 저장해 놓는 임시 기억장치로 메인메모리보다 훨씬 빠른 속도로 작동하는 고속 메모리의 일종입니다. 요즘 들어서는 캐시가 CPU 내부에 설치되기도 하지만 그렇더라도 CPU 와 캐시는 별도의 회로를 독립적으로 형성하고 있습니다.

레지스터 란?
한편 레지스터란 CPU 구조의 일부로써 CPU 에서 연산(계산)에 사용하는 데이터를 기억하는 소규모 기억장치를 말합니다. 컴퓨터는 캐시로 부터 데이터를 읽어들여 레지스터에 저장한 다음 레지스터 사이로 데이터를 전달하면서 연산을 수행합니다.

예를 들어 보겠습니다. TV 의 요리 프로그램을 상상해 보십시오. 요리를 할 때 요리에 들어가는 고기나 야채같은 재료들은 냉장고에서, 간장, 고추장 등 양념들을 싱크대 장에서 그때 그때 꺼내 쓰게되면 번거러울 뿐 아니라 시간도 많이 걸립니다. 그렇다고 모두 꺼내놓고 요리를 하자면 주방이 커져야 합니다. 따라서 TV 요리 프로그램을 보면 적당한 접시에는 재료들을, 작은 종지에는 양념들을 미리 모두 담아놓습니다. 이 접시나 종지가 바로 캐시의 역할을 하는 것입니다. 부엌을 크게 만들어서 모든 것을 다 꺼내 놓을 수 있다면 좋겠지만 그러자면 비용이 많이 들고, 그렇다고 필요한 것을 그때 그때 꺼내 쓰자니 시간이 많이 드니까 위와 같은 방법을 쓰는 것 처럼 컴퓨터에서는 전부 캐시 메모리급으로 메인메모리를 만들자니 돈이 많이 들고, 느린 메인 메모리만 쓰자니 속도가 안 따라주고 하니까 캐시를 도입한 것입니다. 다시 요리로 되돌아가서 요리의 경우는 요리사와 칼, 도마가 CPU 의 구성요소가 됩니다. 요리사는 CPU 의 중앙처리회로, 칼은 연산회로, 도마는 레지스터가 된다고 볼 수 있습니다. 접시에 놓여진 재료들이 요리가 되기 위해서는 반드시 도마위로 옮겨져서 요리사가 휘두르는 칼을 거쳐가야 합니다. 그런데 도마에는 보통 한번에 한가지 재료만이 올라옵니다. 그리고 접시는 재료를 일시적으로 보관하는 기능이지만 도마는 재료를 처리하기 위해 올려 놓을 뿐 처리가 끝난 재료는 다른 그릇으로 옮겨져 보관되고 도마에는 보관의 개념이 없습니다. 레지스터 역시 마찬가지입니다. 레지스터에 들어오는 데이터는 처리되기 위해서 들어오는 것이지 저장되기 위해서 들어오는 것이 아닙니다. 즉, 캐시는 저장을 위한 요소로 CPU 의 보조 구성요소이지만 레지스터는 데이터 처리를 위한 구성요소로써 CPU 의 필수 구성요소입니다.
