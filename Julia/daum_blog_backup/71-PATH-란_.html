* PATH 란?

쉘 상에서, 어떤 Foo.exe 프로그램을 실행하기 위해서 Foo 라는 명령을 주어도, 현재 위치한 폴더 안에 Foo.exe 가 없다면

실행되지 않는다. foo.exe 가 있는 폴더로 가서 Foo 를 입력해줘야한다. 하지만 매번 .exe 파일을 실행시킬 때마다

그 폴더로 이동할 수는 없지 않은가? 그렇다고 Foo.exe 라고 명령을 입력할 때마다 전체 시스템의 폴더를 모두 뒤지라고

할 수는 없는 노릇이다. 하지만 특정 디렉토리 몇 개만 뒤져보라고 지정해준다면? 항상 빠르게 .exe 파일을 실행할 수 있을 것이다.

그 특정 디렉토리의 위치가 바로 PATH 이다. 이 PATH는 파일이 아닌, 환경 변수라 하는 메모리 변수에 저장된다.

따라서 배치 파일이나 쉘 스크립트를 만든 뒤에 이 PATH 경로 안에 넣어주면, 언제어디서나 그 파일을 실행할 수 있는 것이다.

이 PATH 는 환경변수에 등록되어 있으므로, 어떤 명령을 내리면 쉘은 이 환경변수를 참조하여 PATH의 위치를 알아내고, 그 위치을

모두 뒤져서 내가 입력한 명령어를 실행하는 것이다.
