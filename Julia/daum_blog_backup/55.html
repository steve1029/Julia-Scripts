160928 뜬금없이 testcluster 의 apache2 가 제대로 작동하지 않음

리눅스/시행 착오 노트

2016-09-28 19:05:25

바로 어제 UEFI로 데비안을 깔았던 앤데 왜 apache2 가 제대로 작동하지 않는지 모르겠다

apache2 mysql-server php5 php5-mysql

4개를 전부 다시 지우고 다시 깔고 재부팅하고 이래도 안됨. mediawiki ganglia 모두 접속이 안된다.

http://163.152.40.36/ 으로 들어가면 아무것도 안나오고 접속이 안된다고 뜸.

어디가 문제일까...


# /etc/init.d/apache2 status


를 하니까 could not reliably determine the server's fully qualified domain name 어쩌구 이런 에러가 떠서


# vi /etc/apache2/apache2.conf


의 마지막 줄에


ServerName localhost


를 추가하고 저장하고 


# /etc/init.d/apache2 restart

# /etc/init.d/apache2 status


하니까 사라지긴 했는데, 그래도 여전히 서버로 접속은 안된다.


데비안의 웹 포트는 80번인데,


# netstat -ntl 


하면 80번은 분명 열려있음. 왜 접속이 안됟까?
