DLL 이란?

DLL 을 알려면 먼저 라이브러리란 무엇인지 알아야 한다.

코딩을 하다 보면 자주 쓰는 함수가 있는데, 그 때마다 함수를 만들어 쓰기는 매우 귀찮고 어려운 일이다.

그래서 자주 쓰는 함수를 따로 모아서 파일을 만들어 둔 것이 라이브러리이다.

마치 python 의 module 이라고 생각할 수도 있겠지만 살짝 다르다.

python 의 module 은 C 언어의 header 파일에 대응 되며

둘다 코드로 이루어져 있다.

그러나 라이브러리는 코드가 이미 컴파일 된, 바이너리(2진수) 파일이라서 사람이 읽을 수 없다.

라이브러리를 다시 원래 코드로 돌려 놓는 작업을 해야 읽을 수 있다. 실제로 가능하기도 하고.

어찌 됐든 라이브러리가 뭔지는 대충 알았다.

이제 static link 와 dynamic link 에 대해서 알아보자.


Static link



그럼 어떤 코드가 라이브러리에 있는 함수를 이용할 때, 이 라이브러리에 있는 함수 내용을 그대로 복사해서

컴파일 할 수 있다 . 이를 static link 라 한다.


Dynamic link


어떤 코드를 컴파일 할 때, 라이브러리 내에서 2진수로 쓰여진 함수들이 복사되는 것이 아니라 단지 호출하기만 하는 방법이다. 코드 파일에서 라이브러리를 호출하는 코드가 적혀있다면, 컴파일 시에 실행 파일에는 라이브러리의 위치에 대한 정보가 기억된다. 그래서 프로그램이 실행되면 dll 파일도 같이 메모리에 올라가게 되며, 프로그램이 dll 내의 함수를 호출하게 된다.


DLL 의 장점

①     한 코드를 여러 프로그램이 동시에 사용하기 때문에 메모리가 절약된다.

②     정적링크를 사용하는 경우 실행 파일에 라이브러리의 함수가 모두 포함되어 실행파일이 커지지만

DLL을 사용하는 프로그램은 크기가 작다.

③     DLL을 교체하여 프로그램의 성능을 향상시키기 쉽다.

④     리소스의 교체가 가능하다.

⑤     코드의 양이 적어지므로 디버깅이 용이해진다

⑥     혼합 프로그래밍이 가능해 진다

⑦     프로그래머끼리 분담 작업이 용이하며 재사용성도 뛰어나다.
