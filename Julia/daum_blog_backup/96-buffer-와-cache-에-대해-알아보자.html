컴퓨터는 기본적으로 RAM 과 CPU 사이에서 데이터가 오가면서 일을 한다.


RAM에 데이터를 올리고, 이를 CPU에 보내고, CPU가 데이터를 받아 처리하고, 결과를 다시 RAM에 저장하는 방식으로 돌아간다.


하지만 RAM은 전원이 차단될 시 데이터가 모두 사라지는 단점이 있고, 가격대가 높은 편이라 용량과 데이터의 수명에 한계가 있다.'


따라서 등장한게 HDD 다. HDD 는 전원이 차단되어도 데이터를 그대로 보존할 수 있다. 또한 RAM 과는 다르게 가격이 싸지만 느리다. 


그래서 컴퓨터의 데이터 처리 방식은 다음과 같이 변한다.


HDD -> RAM -> CPU


HDD <- RAM <- CPU


전원을 켜면 HDD에 저장된 정보가 RAM에 올라가고, CPU와 RAM 사이에 데이터 처리가 일어나며,


저장할 시 RAM의 내용이 다시 HDD 에 옮겨간다.


하지만 또다른 문제가 발생했는데, CPU 의 데이터 처리 속도는 점점 빨라짐에도 각 기기간의 데이터 전송 속도가 이를 따라가지 못해


실질적인 처리 속도는 그다지 나아지지 않게 되었다. 그래서 데이터를 빨리 전송할 수 있는 방법이 나왔는데, 그 중 대표적인 것이


buffer 와 cache 이다.

cache 메모리


캐시 메모리는 cpu 에 가장 빈번히 전달 되어야 할 명령어 및 데이터를 원활히 전송할 수 있도록 제작된 고속도의 특수 메모리를 말한다. 쉬 메모리안에 보전되어 있는 정보 및 데이터는 컴퓨터에 주 기억 장치에 저장 되어 있을 때 보다 약 10배는 빠른 속도로 접속될 수 있다. 처리장치가 캐쉬 메모리로부터 접속할 수 있는 명령어와 데이터의 양이 많으면 많을수록 전체적인 컴퓨터의 작동 속도는 빨라지는 것이다. 


캐쉬 메모리의 레벨 종류 

일반적으로 캐쉬 메모리에는 두가지 종류의 레벨이 있다. 하나는 CPU칩안에 내장되어 있는 내부 캐쉬이고 또 다른 하나는 일반적으로 시스템 보드 위에 내장되어 있는 외부 캐쉬이다. 내부 캐쉬는 기본 캐쉬 혹은 레벨1(Level 1) 캐쉬라고도 불리우며 외부캐쉬는 2차 캐쉬, 혹은 레벨2(Level 2)라고도 일반적으로 불리어 진다. 대부분의 데스크탑 컴퓨터에 있어 내부 캐쉬 메모리는 그 크기에 있어 1KB에서 32KB(1000byte에서 32000byte) 범위 안에서 분류가 된다. 이와 대조적으로 외부 캐쉬 메모리의 구성은 일반적으로 이보다 훨씬 커서 64KB에서 1MB(64000byte에서 1000000byte)범위 안에서 분류가 된다. 일반적으로 사람들이 업그레이드 캐쉬라 말하는 것은 이 외부 캐쉬를 말하는 것이다. 업그레이드용 외부 캐쉬 메모리는 시스템 보드 위에 있는 소켓에 각각의 캐쉬부품을 연결해야 하거나 캐쉬 모듈을 확장용 캐쉬 전용 소켓에 연결해야 하는 경우가 있다. 이와 달리 대부분의 경우에 있어서의 업그레이드용 내부 캐쉬 사용을 위해서는 CPU자체를 바꿔야 하는 경우가 많다. 

참조 : 286과 386체제와 같은 몇 몇 초기세대의 개인용 컴퓨터의 경우 CPU칩에 내부 캐쉬 메모리가 내장되어 있지 않다. 이러한 경우(만약 내장되어 있다면) 외부 캐쉬가 기본 캐쉬로 사용될 것이고 레벨 1 캐쉬가 될 것이다. 


캐쉬 메모리는 어떻게 작동하는가?
캐쉬 메모리라는 것에 대해 좀 더 흥미로운 시각에서 살펴보자면, 당신이 주문하는 음료수 종류만 정확히 맞춰 서빙해주는 파티를 생각하면 될 것이다. 여기서 음료수는 데이터에 해당될 것이고, 파티장 뒤 쪽에 있을 매점은 주기억 장치, 음료수를 꺼내는 냉장고는 캐쉬 메모리인 셈이다. 만약 파티를 하는 사람 중 누군가가 다이어트 펩시를 주문했다면 파티 주인은 먼저 냉장고에 가서 다이어트 펩시가 있나 살펴볼 것이다. 냉장고에 있다면 바로 손님께 서빙될 것이고, 그렇지 않다면 뒤 쪽 매점으로 가서 펩시를 구해올 것이다. 물론 이 경우 시간은 더욱 오래걸리기 때문에 파티 주인은 다음부터는 한 번에 여섯 들이를 구입해서 냉장고에 넣어놓을 것이다. 이러한 경로를 통해 대부분의 경우 그 다음에 같은 걸 주문하는 사람은 냉장고로부터 바로 서빙받을 수 있을 것이다. 같은 방식으로 캐쉬 제어기(cache controller)가 주기억 장치로부터 명령어를 검색할 때 이를 통해 얻은 몇가지 관련자료를 바로 캐쉬 메모리로 보내는 것이다. 이런 경로를 통해 CPU에서 같은 명령어를 찾고자 할 때 이 정보는 캐쉬에 이미 저장되어 있는 것이다.(CPU에서 찾고자 하는 자료가 캐쉬에 이미 저장되어 있을 때 캐쉬 메모리 적중(cache hit)의 수치적 형태로 나타날 것이다.)



캐쉬는 어느 정도 보유하고 있어야 하나?

일반적인 100MHZ 인텔 마더보드의 경우, 주기억 장치로부터 모든 정보를 입수하는 데 시간이 180시간이 걸린다면 캐쉬 메모리부터 전송받는 데는 이에 비해 약 45시간 정도 걸린다고 보면 된다. (이는 정보 검색, CPU의 정보 요청, 정보 검증, 그리고 데이터 접속 시간을 모두 포함했을 때의 경우이다.) 캐쉬 메모리를 이용한 정보처리 속도에 있어서의 엄청난 장점을 통해 볼 때, 주기억 장치에서 전달되는 모든 정보를 캐쉬에 입력을 하는 것이 맞을 것이나, 일반적으로 캐쉬 메모리는 주기억 장치가 쓰는 DRAM칩에 비해 가격대가 6배나 비싼 SRAM(Static Ram) 칩을 사용해야 한다는 단점이 있다. 따라서 컴퓨터 시스템에 많은 양의 캐쉬를 이용하는 것은 비용 효과 측면에서 좋지가 않다. 앞 서 비교한 파티를 이용해서 다시 본다면 이는 마치 매점에서 파는 모든 종류의 음료수를 전부 다 사와 냉장고에 쌓아두는 것과 같다고 볼 수 있다. 결국 냉장고 하나를 이용해 시간 절약과 편리성을 도모한다 하더라도 이렇게 되면 결국 파티장 뒤 쪽에 있는 매점은 투자한 쓸모가 없어져 버리는 셈이 된다. 캐쉬 메모리의 경우도 이와 같다 생각하면 될 것이다. 자주 검색 요청이 이루어지는 명령어만 256K 정도의 용량을 가진 캐쉬에 저장해 둔다면 처리 속도에 있어 엄청난 시간 절약을 해 줄 것이지만 그렇다고 여기에다 256K를 더해 512K를 만든다고 해서 그 이상 컴퓨터의 처리속도가 빨라지지는 않는다는 것이다. 


관련 용어 설명 

Asynchronous SRAM - 클럭 신호가 제어 신호와 맞물리지 않아도 되는 SRAM의 종류로서 Synchronous SRAM과 가격대 비교면에서 약 30% 정도가 싸다.

버스트(Burst) - Synchronous cache 메모리 종류의 하나로서 Asynchronous 캐쉬 메모리보다 30내지 50% 정도 처리속도가 빠르며 가격은 50%정도 더 비싸다. 

캐쉬 제어기 - CPU와 캐쉬, DRAM 제어기 사이의 인터페이스(interface)를 제어하는 회로 

캐쉬 적중 - CPU에서 요청한 정보의 위치가 캐쉬에서 발견된 경우

캐쉬 미스(cache miss) - CPU에서 요청한 정보의 위치가 캐쉬에서 발견되지 않은 경우

CELP 소켓 - Card Edge Low Profile(카드 에지 로우 프로파일)의 약자로서 일반적으로 캐쉬 모듈용의 소켓을 말한다.

코우스트(COAST) - Cache on A Stick(캐쉬 온 어 스틱)의 약자로서 소비자들에게 인기있는 캐쉬 모듈에 대한 상세 설명도를 말한다. 

직통 캐쉬(Direct-mapped cache) - 각각의 데이터 입구에 하나의 연결통로만이 존재하는 캐쉬

외부 캐쉬 - 컴퓨터 처리장치 외부에 존재하는 캐쉬로서 프로세서 가까이에 위치한 시스템 보드 위에 바로 납땜이 되어 있거나 처리장치 근처에 캐쉬 모듈의 형태로 소켓에 접속되어 있다. 

종합 연결 캐쉬(Full-associative cache) - 중앙 기억장치가 모든 캐쉬라인으로 연결될 수 있도록 하는 캐쉬의 기능 

목차(Index) - 캐쉬에 내장되어 있는 자료의 정확한 위치를 찾기 위한 CPU의 주소 메모리 비트의 부분적 집합

내부 캐쉬 - 일반적으로 CPU 칩 안에 위치하는 캐쉬의 형태 

레벨 1(L1) - 처리장치에 가장 가까이 위치한 캐쉬로서 일반적으로 CPU 칩안에 내장되어 있으며 기본 캐쉬라고 일컬어지기도 한다. 

레벨 2(L2) - 처리장치에 2번째로 가까이 위치한 캐쉬로서 일반적으로 시스템 보드위에 위치하고 있으며 2차 캐쉬라고 일컬어지기도 한다. 

파이프라인 버스트(Pipeline Burst) - Burst보다 약간 가격이 저렴한 Synchronous 캐쉬로서 버스트와 비슷한 기능을 수행한다. (버스트가 처리 속도에 있어 더 빠르긴 하나 실제적으로 시스템 보드는 이에 그다지 큰 영향을 받지 않는다.) 

기본 캐쉬 - 레벨 1 캐쉬와 동일 

2차 캐쉬 - 레벨 2 캐쉬와 동일 

캐쉬내 연결 집합(Set-associativity) - 캐쉬 내에서 중앙 기억 장치로의 단일 연결 통로를 만들어 놓은 곳 

SRAM - Static Random Access Memory(스테이틱 랜덤 엑세스 메모리)의 약자. 캐쉬 모듈 위에서 사용되는 메모리칩으로 DRAM보다 속도가 빠르고 가격이 비싸다. 

Synchronous SRAM - 클럭 신호가 제어 신호와 맞물려야 하는 SRAM의 종류로서 cache 메모리와 CPU간의 경로를 밀착시키는 역할을 한다. 버스트와 파이프라인 버스트의 두가지 종류가 있
다.

태그(Tag) - CPU의 주소 메모리 비트의 부분적 집합으로, 연결된 중앙 기억 장치의 위치를 캐쉬 디렉토리 란의 태그 비트를 통해 확인하는 데 이용된다. 

태그 램(Tag Ram) - 캐쉬는 일반적으로 두가지 섹션으로 나뉜다. 이 중 Tag Ram 섹션은 캐쉬 안에 내장되어 있는 데이터 위치를 나타내는 태그 주소를 저장하는 역할을 한다. 이것은 실질적 데이터나 명령어를 저장하는 데이터 RAM 섹션보다 작은 크기를 가지고 있다. 

재기록(Write Back 혹은 Copy Back) 작업 - 캐쉬에 기록된 데이터는 캐쉬 내부의 관련 라인이 생성 될 때까지 중앙 기억 장치에 기록이 되지 않는다. 

직접 기록(Write Through) - CPU로부터 전송받은 데이터를 정보 연결의 일관성을 위해 바로 cache와 중앙 기억 장치에 기록하게 하는 기술

버퍼와 캐쉬의 차이
1. 버퍼 : Buffer(속도완충기)
일시적인 데이터 저장소로 이용하는 기억 장치로 (서버과 클라이언트의 속도차 때문에 서버가 
버퍼에 내용을 저정해 놓으면 클라이언트는 버퍼의 내용을 가져간다.)

2. 캐쉬 : Cache(속도 완충기)
속도가 빠른 중앙처리장치와 상대적으로 느린 ram사이의 원활한 데이터 통신을 위해 주기억장치의 정보를 일시적으로 저장하는 고속기억회로
버퍼와 같은 개념이지만 인공지능이라는 점에서 속도가 훨씬 빠르다

종류 : L1캐시, L2캐시, 웹브라우저 캐시
L1캐시 : cpu내에 내장되어있는 캐시 메모리
L2캐시 : 메인메모리에 비해 더욱 빠르게 접근할 수 있도록 cpu에 내장하거나 메인보드에 내장하는 메모리입니다.
웹브라우저캐시 : 방문한 웹사이트의 파일을 C:WINDOWSTEMPORARY Internrt Filie폴더에 저장하여 사용자가 다시 접속했을때 서버에 데이터를 전송 받는 것이 아니라 로컬하드디스크에서 로드하여 속도를 끌어 올리는 역할
모든 브라우저는 기본적으로 제공한다.

캐시(cache)메모리는 속도가 다른 두 장치 사이에 위치하는 임시기억 장소로 장치간의 데이터 엑세스 속도를 향상시키는 구실을 하면서 반복적인 엑세스에서 처리 속도를 향상시킵니다. 

캐시메모리는 일종의 밀어내기식 구조로 되어 있어서 한번 들어온 데이터는 다른 데이터에 의하여 밀려나거나 컴퓨터가 꺼지지 않는 이상 계속 유지되기 때문에 반복적인 엑세스를 할 수 있습니다.

버퍼(buffer)메모리도 두 장치 사이에 속도 차이가 날때 사용하지만(CPU와 주변장치 사이에 발생하는 전송속도의 차이를 해결해 주는 임시 기억장치) 속도 향상을 위한 것이 아니라 빠른 쪽에서 데이타가 느린 쪽으로 보내질 때 데이터의 손실을 막기 위해서 쓰입니다.

버퍼메모리는 데이터의 처리속도나 처리단위, 데이터 사용시간이 서로 다른 두 장치나 프로그램 사이에서 데이터를 주고받기 위한 목적으로 사용되는 장치로, 속도 차이가 나는 하드웨어나 우선순위가 다른 프로그램의 프로세스들에 따라 공유하다가 각각의 장치나 프로세스가 상대방에 의해 정체되지 않고 제대로 움직일 수 있도록 해주면서 느린 시스템의 구성요소가 동작을 기다리는 동안 유용하게 사용될 수도 있고, 논리회로에서 신호의 전달을 지연시키는 역할도 합니다. 따라서 버퍼메모리의 양이 크면 클수록 전송속도를 높일 수 있습니다. 이 때문에 중앙처리장치보다 속도가 늦은 하드디스크, CD-ROM, 광자기 드라이브나 프린터, 디지털카메라 등에는 대부분 버퍼메모리가 장착되어 있습니다.

버퍼메모리는 한마디로 데이터를 전송하는 상호간의 장치에서 고속의 장치와 저속의 장치 간의 속도 차이로 인해 저속의 장치가 작업을 추리하는 동안 고속의 장치가 기다려야 하는 현상을 줄여주는 기술로 버퍼메모리를 사용하지 않는다면 컴퓨터는 프린터가 인쇄를 끝낼 때까지 아무런 작업을 할 수 없게 되지만 버퍼메모리를 사용하면 컴퓨터에서 전송한 데이터가 버퍼메모리에 저장되어 프린터는 컴퓨터로부터 집적 데이터를 받는 것이 아니라  버퍼메모리에서 데이터를 조금씩 읽어서 출력하므로 컴퓨터는 버퍼메모리로 전송작업을 마치고 프린터가 출력을 하는 동안 다른 작업을 할 수 있게 되는 것입니다. 

버퍼메모리는 각 장치나 프로세스가 상대방에 의해 정체되지 않고 잘 동작할 수 있도록 해주므로 "데이터의 중간 저장소"라는 측면에서 버퍼메모리는 캐시메모리와 같지만, 캐시메모리가 어떤 작업의 속도를 증진시키기 위해 존재하는데 비해, 버퍼메모리는 개별 작업들 간의 협동(개별 작업들 사이에서 데이터 주고받기)을 지
