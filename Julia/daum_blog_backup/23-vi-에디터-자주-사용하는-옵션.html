vi 에디터 자주 사용하는 옵션

리눅스/vi 사용법

2016-09-26 10:00:29

리눅스에서 특정 사용자의 vi 에디터 옵션 파일은

 

/home/사용자/.vimrc 파일로 존재한다.

 

루트권한자에게 vi syntax 를 적용하려면 /root 에 .vimrc 파일을 만들어주면 된다.
전체사용자에게 적용하고 싶다면
/etc/vimrc 혹은 /etc/vim/vimrc 파일을 수정하면 된다. 우분투 버전에 따라 다르니
확인하고 수정하면 된다.

 

리눅스 설치하자마자 aptitude로 필수 애플리케이션을 설치하고 난뒤,

 

vi 설정 파일을 이 위치에 옮겨 놓도록 하자. config 파일들을 좀 더 쉽게 수정할 수 있다.

 

다음 포스트도 참고해보자.https://github.com/johngrib/simple_vim_guide/blob/master/md/vimrc.md

 

set nu

set smartindent

set number                 " line 표시를 해줍니다.

set ai                       " auto indent

set si                       " smart indent

set cindent                 " c style indent

set shiftwidth=4            " shift를 4칸으로 ( >, >>, <, << 등의 명령어)

set tabstop=4               " tab을 4칸으로"

set ignorecase             " 검색시 대소문자 구별하지않음

set hlsearch                " 검색시 하이라이트(색상 강조)

set expandtab              " tab 대신 띄어쓰기로

set background=dark      " 검정배경을 사용할 때, (이 색상에 맞춰 문법 하이라이트 색상이 달라집니다.)

set nocompatible          " 방향키로 이동가능

set fileencodings=utf-8,euc-kr    " 파일인코딩 형식 지정

set bs=indent,eol,start    " backspace 키 사용 가능

set history=1000            " 명령어에 대한 히스토리를 1000개까지

set ruler                     " 상태표시줄에 커서의 위치 표시

set nobackup               " 백업파일을 만들지 않음

set title                      " 제목을 표시

set showmatch            " 매칭되는 괄호를 보여줌

set nowrap                 " 자동 줄바꿈 하지 않음

set wmnu                   " tab 자동완성시 가능한 목록을 보여줌

syntax on                   " 문법 하이라이트 킴"

각 옵션들에 대한 설명은 다음과 같다.

 

- ts(tap stop) : 문서에 있는 '\t'문자를 몇 칸으로 보여줄지 결정하는 변수

- sw(shift width) : >> 또는 << 키로 탭 사이즈를 조절할때나 인덴트에서 자동 줄맞추기를 할때 사용되는 화면 표시 크기

- sts(soft tab stop) : Tap 키를 눌렀을 때 표시되는 간격을 지정해 주는 변수

 

○ cindent

: C, C++, JAVA 스타일의 자동 들여쓰기 활성화.

  블럭의 처음 시작 및 라인을 바꿀 때 자동 들여쓰기를 해준다.

 

○ smartindent

: cindent 와 비슷 하지만 #if, #ifdef, #define 등과 #으로 시작하는 전처리기 명령어에 대해서는 자동 들여쓰기를 무시하고 다음 라인의 첫번째 열로 이동한다.
