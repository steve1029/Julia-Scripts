blocking 과 non-blocking 의 차이

컴퓨터 언어/Python

2017-04-10 15:43:16

mpi4py 기준으로 설명하겠다.


from mpi4py import MPI


comm = MPI.COMM_WORLD # comm is abbreviation of communicator


1. Blocking Communication


comm.Send를 이용해서 데이터를 보내면, receive buffer 가 채워지면서 '받았다' 는 신호를 보낸다.

이 신호를 받아야지만 comm.Send 메쏘드가 끝나고 다시 return 한다. 즉 communication이 끝나기 전까지

return 하지 않기 때문에 blocking 이라고 한다. comm.Recv 메쏘드 또한 receive buffer가 채워졌다는 신호를 되돌려받기 전까지

return 하지 않으므로 전송이 마무리 되지 않으면 계속 기다리는 상태가 된다.


2. Non-blocking Communication


comm.isend , comm.Isend

comm.irecv, comm,Irecv


immediate 의 i 가 추가된 naming 이다.

버퍼가 채워졌든가 말든가 일단 보내고 자긴 다른 일 계속한다. 그래서 중간에 메세지 전송이 끝나기 전까지 기다리는 blocking 현상이

발생하지 않는다. 그래서 데이터 전송과 데이터 처리의 overlapping 이 일어나 전체 작업 시간이 단축될 수 있다. 하지만 코드의 안정성이

떨어질 수 있어서, 최적화 단계에서 쓰여야 하는 방법이다.
