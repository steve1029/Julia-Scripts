1. 1차원 배열


1차원 배열을 선언해보자.


int ar[20]={0,};


int 4개가 들어갈 수 있는 배열을 선언하고 0으로 모두 초기화하였다. 이때 ar 은 ar[0] 가 저장된 주소를 가리키는 포인터이다. ar[0] 는 선언과 동시에 메모리의 한 부분을 차지하게 되므로, 정해진 값이다. 즉 ar 은 포인터 변수가 아닌 포인터 상수이다.


int ar[3] = {1,2,3};

int ar2[3];

ar2=ar // 불가


배열명은 포인터 상수이므로 좌변값에 올수 없으며 값을 바꿀 수도 없다. 이미 메모리의 한 곳에 저장된 것을 임의로 옮기려면 다른 방법을 써야한다.


char str[20];

scanf("%s", str);


str 자체가 이미 포인터이므로 scanf 함수에서 인자로 줄 때 & 가 필요없다. 어떤 변수를 인자로 주려면 &변수 꼴로 입력해야 했음을 기억하라.


초기화 방법


int ar[1000] = {1,2,3,4};


이렇게 배열의 크기보다 적게 초기화시켜주면 나머지 값은 전부 0으로 초기화한다.


int ar[1000] = {0};

int ar[1000] = {0,};


이런 식으로 응용해줄 수 있다.

또한 배열의 크기를 지정해주지 않고 딱 맞게 초기화해줄 수도 있다.


int ar[] = {1,2,3,4};


컴파일러는 이걸 보고 자동으로 크기가 4 인 배열을 만들어준다.


2. 2차원 배열

2차원 배열을 초기화 하는 방법을 알아보자.


int array[3][3] = {{1},{2,3},{3,3,3}};

int array[3][3] = {1,2,3,4,5};

int array[][2] = {{1,2,3},{1,2,3}};


위 경우는 딱히 설명을 안할테니 유추해보길 바란다. 


마지막의 경우 배열의 크기를 생략하는 방법인데, 1차 크기만 생략 가능하고 2차 크기는 반드시 넣어줘야 한다. 

그도 그럴 것이 둘 다 없으면 어떤 형태의 배열을 만들지 컴파일러가 모를테니까. 

그리고 살펴보면 행이 2개고 열이 3개인 배열로 초기화해줬는데 선언할 때는 열이 2개라고 선언했다. 이럴때 에러가 발생하지 않을까? 

그렇지 않다. 하나가 비어있으면 컴파일러는 나머지 하나가 행인지 열인지 초기화 배열을 보고 알아서 판단한다. 

이 경우 [2] 는 뒤에 있긴 하지만 컴파일러는 행이 2개라는 뜻으로 알아듣는다.
