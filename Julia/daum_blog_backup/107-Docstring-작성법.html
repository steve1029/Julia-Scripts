보통 스크립트의 첫 부분이나 함수의 첫 부분에 docstring 을 작성하게 된다.


convention 은 다음과 같다. convention 을 따르는 것이 좋다. 제일 알아보기 쉽고 간결한 형식이라 제일 많이 쓰이는 방법들이


convention 으로 굳어진 것이기 때문이다.


다음의 python 공식 홈페이지의 설명을 참조하라. 4.7.6 의 Documentation strings 항목을 참조.


https://docs.python.org/2/tutorial/controlflow.html#arbitrary-argument-lists


첫 문장은 프로그램이나 함수에 대한 간략한 설명. 반드시 대문자로 시작하도록 한다. 요약문이므로, 되도록 한 문장으로 마친다.


부가적인 설명은 한 줄을 띄우고 설명하도록 한다. 요약문과 부가 설명을 구별하기 위해서 반드시 한 줄을 띄우고 적도록 하자.


docstring 의 시작과 끝은 """ 으로 마무리해야한다. 그래야 다음 method 가 먹힌다.


>>> def my_function():
...     """Do nothing, but document it.
...
...     No, really, it doesn't do anything.
...     """
...     pass
...
>>> print my_function.__doc__
Do nothing, but document it.

    No, really, it doesn't do anything.


>>> my_fuction.__doc__


드디어 이 함수가 무엇을 호출하는지 알 수 있게 되었다.


함수의 결과물을 보라. 첫 번쨰 문장은 """ 바로 뒤에 오며, 출력될 때 indentation이 안들어가 있지만,


convention 에 따라 작성한 부가 설명란은 indentation 이 들어가 있어 요약문과 확실히 구별되어 출력된다.


한마디로, convention 을 지켜라!
