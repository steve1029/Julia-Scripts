https://docs.continuum.io/anaconda/install/linux


아나콘다 공식 홈페이지에서 리눅스용 아나콘다 설치파일을 yboot 로 다운 받는다.


윈도우에 받아서 xftp 로 yboot의 /nfsroot/y101/root 로 전달하자.


우리는 각 노드의 루트 파일과 홈 서버에 anaconda 를 설치해줄 것이다.


# chroot /nfsroot/y101


그리고 설치파일을 실행하자.


# bash /root/Anaconda2 어쩌구저쩌구.sh


엔터를 치다보면 설치 경로를 물어본다.


# /ananconda2 에 설치하자. /root 에 설치하는 것은 안된다. yboot 의 /root 는 모든 노드가 공유하지만,


그 말인즉 yboot 의 아나콘다를 이용해서 계산한다는 뜻이기 때문에 매우 위험하다.


설치를 다 마치면 마지막에 질문을 한다.


“Do you wish the installer to prepend the Anaconda<2 or 3> install location to PATH in your /home/<user>/.bashrc ?”


우린 당당하게 no 라고 한다.


설치를 마쳤으면 다음 파일을 수정해야한다.


# vi /etc/bash.bashrc


여기에 다음 항목을 추가한다.


export PATH="/anaconda<2 or 3>/bin:$PATH"

export LD_LIBRARY_PATH=/usr/local/lib


이는 환경변수에 아나콘다의 라이브러리를 추가해주는 작업이다. 각 사용자의 ~/.bashrc 에 등록해도 되지만,


우린 모든 사용자가 아나콘다를 쓰게 하고 싶으므로 이렇게 한다.


그리고 다음을 실행해주자.


# source /etc/bash.bashrc


새로운 설정을 적용하는 방법이다.


이제 chroot 환경을 나오자


# exit


다른 모든 노드의 루트파일에 같은 작업을 해주어야 하므로


# rsync -aHv --delete /nfsroot/y101/ /nfsroot/y102


명령어를 모든 노드에 대해 실행해주자.
