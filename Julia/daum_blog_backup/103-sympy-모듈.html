symbolic calculation 을 가능하게 해주는 모듈.


매스매티카와 같은 계산이 가능해진다. 일반적인 언어에서는 불가능한 symbolic 계산이 파이썬에서는 가능하다.


아나콘다를 사용하거나, aptitude 에서 python-sympy 를 설치하자.


import sympy as sy


먼저 사용할 변수들을 선언해줘야 한다.


x, y, z = sy.symbol('x y z')


이제 이 변수들은 symbol 로서 사용 가능하다.


변수를 선언할 때, 띄어쓰기로 x y z 를 구별하거나 , 를 이용해서 구별해줘야한다.


가령


x,y,z = sy.symbols('xyz')


TypeErrorTraceback (most recent call last)

<ipython-input-5-c29bc8e0acf3> in <module>()

----> 1 x,y,z = sy.symbols('xyz')


TypeError: 'Symbol' object is not iterable


내가 입력한 object가 iterable 불가능 이라고 나온다. iterable 가능한 form 으로 argument 를 입력해줘야 한다.


다음의 코드를 보고 이상한 점을 느껴보자.


>>> x = symbols('x')

>>> expr = x + 1

>>> x = 2

>>> print(expr)

x + 1


왜 이런 결과가 나오는가?

x=2 라 하면 새로운 파이썬 변수 x 를 선언하고 초기화한 것이다. 그러나 이 x 는 sympy 에서 만든 x 가 아니다. 파이쎤 변수와 sympy 변수를 구별해야 한다. 파이썬의 특징 중 하나가, 어떤 변수를 이용한 표현이 있을 때, 변수가 바뀌어도 그 표현이 바로 바뀌지는 않는다. 다음의 예를 보자.

>>> x = 'abc'

>>> expr = x + 'def'

>>> expr

'abcdef'

>>> x = 'ABC'

>>> expr

'abcdef'


이런 식으로, 바로 바뀌지 않는다. 그런데 이럴 거면 sympy 를 쓸 이유가 없지 않은가? 어떤 변수를 마음 대로 바꿔서 함수의 결과를 보고 싶은 것이 symbolic math 인데...?


다 방법이 있다.


>>> x = symbols('x')

>>> expr = x + 1

>>> expr.subs(x, 2)

3


이렇게 하면 symbol의 의미를 바꿀 수 있다.


어떤 수식을 입력하고 싶다면?!


Eq 함수를 사용하자.


sy.Eq(x+1,4)


x+1 == 4


를 입력하면 왜 안될까? == 는 python 에서 두 값이 같은지 테스트 하라는 의미이기 때문이다.


두식이 같은지 테스트 하는 방법은 두 가지가 있다.


>>> a = (x + 1)**2

>>> b = x**2 + 2*x + 1

>>> simplify(a - b)

0


>>> a = cos(x)**2 - sin(x)**2

>>> b = cos(2*x)

>>> a.equals(b)

True


둘 중 아무거나 사용하자.
